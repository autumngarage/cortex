## Context

Today's session followed a ~5-hour Touchstone rename (toolkit → Touchstone, transfer to autumngarage org) and an architecture discussion that converged on a third tool — Cortex — for project memory. The discussion evolved through several candidate shapes:

1. Initially: Atlas (a persistent project map) + Ledger (a chronicle) as two separate tools.
2. Observed: together they're a "brain" — perception + memory.
3. Attempted fold-in: is this a Sentinel feature, not a tool?
4. Settled: it's its own tool because (a) cognitive responsibility is cleanly distinct from Sentinel's executive role, (b) it serves multiple writers (Sentinel, Touchstone hooks, Claude Code, humans), (c) it can stand alone if needed.
5. Pushed on "standalone" — Cortex shells out to `claude` CLI directly, same pattern as Sentinel, no shared provider layer. Answers the "it doesn't stand alone" objection.
6. Biological framing landed: Touchstone = cytoskeleton, Sentinel = metabolism, Cortex = genome + memory.

## What we decided

- Cortex is a file-format protocol with a reference Python CLI. Six layers: Doctrine, Map, State, Plans, Journal, Procedures (see SPEC.md §3).
- Polyrepo: separate from Touchstone and Sentinel, own brew tap (`autumngarage/homebrew-cortex`). Composes by file contract only.
- Spec-first: SPEC.md is the primary artifact (see doctrine/0003).
- Dogfood target ordering: Sentinel first (known code), Touchstone second, sigint third (the project this tool aims to replace manual work on).
- Scope explicitly excludes portfolio / multi-project views (the Lighthouse conversation) — one project at a time.
