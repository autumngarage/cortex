> Cortex v0.3.0 ships the write-side authoring surface — `cortex journal draft <type>`, `cortex plan spawn <slug>`, the new T1.10 release trigger + `release` journal template, and a SPEC § 4.2 orphan-deferral check in `cortex doctor`. The CLI is now a two-way tool for any project on the v1.0 path; first release on the post-rerank `plans/cortex-v1` arc.

## What shipped

Five PRs, each a v0.3.0 work-item from [`plans/cortex-v0.3.0`](../plans/cortex-v0.3.0.md):

- **PR #44 — T1.10 release trigger + `release` journal template** ([feat: T1.10 release trigger…](https://github.com/autumngarage/cortex/pull/44)). Adds the tenth Tier-1 Protocol trigger ("a tagged release / distribution artifact shipped") with a matching `journal/release.md` template that captures `## Downstream docs this changes` as the seed for v0.5.0's `--audit-instructions`. SPEC bumped to v0.4.0-dev (additive minor per § 7), Protocol to v0.2.1. `cortex doctor --audit` now walks `git tag --list` for tag-sourced T1.10 fires alongside the existing commit walk.
- **PR #45 — `cortex journal draft <type>`** ([feat: cortex journal draft…](https://github.com/autumngarage/cortex/pull/45)). The keystone authoring command. Resolves templates project-first then bundled-fallback, pre-fills date and (with `--title`) the H1, appends an auto-context comment block (recent `git log` + `gh pr view` output, gracefully degrading when `gh` is missing/unauth/no-PR). Default flow opens `$EDITOR`; `--no-edit` writes directly. Refuses to write to incompatible stores per SPEC § 7. Used to author this very release entry.
- **PR #46 — `cortex plan spawn <slug>`** ([feat: cortex plan spawn…](https://github.com/autumngarage/cortex/pull/46)). Scaffolds an active Plan with seven-field frontmatter (Status, Written, Author, Goal-hash computed from `--title`, Updated-by seeded, Cites) and the five required SPEC § 3.4 sections. The bundled plan template was tightened so placeholder text honestly fails `cortex doctor`'s measurable-criteria + grounding-citation checks rather than passing by accident on magic strings.
- **PR #47 — `cortex doctor` orphan-deferral check** ([feat: cortex doctor orphan-deferral…](https://github.com/autumngarage/cortex/pull/47)). SPEC § 4.2 is now structurally enforced: every active Plan's `## Follow-ups (deferred)` bullet must cite a Plan, Journal entry, or Doctrine entry that *actually exists* under `.cortex/`. Rejects self-citation and dangling references. SPEC § 4.2 itself was expanded to formalize Doctrine as a valid resolution target (the practical "out of scope per Doctrine N" pattern this repo already used).
- **PR #48 — v0.3.0 release prep (this PR).** Version bump 0.2.7 → 0.3.0 in `__init__.py` + `pyproject.toml` + `uv.lock`, this release journal entry, plan closure (cortex-v0.3.0 → `Status: shipped`), parent-plan tick-offs in `plans/cortex-v1.md` v0.3.0 sub-section.

Spec / Protocol versions accepted: SPEC v0.3.x and v0.4.x; Protocol v0.2.x. The dev-stamp on SPEC v0.4.0-dev stays until further v0.4.0–v0.6.0 iterations land; SPEC freezes ceremonially at v1.0.0 per [`plans/cortex-v1`](../plans/cortex-v1.md).
