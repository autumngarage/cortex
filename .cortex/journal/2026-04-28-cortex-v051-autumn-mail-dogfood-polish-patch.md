## What shipped

PR #68 — `cortex next` filters `{{ placeholder }}` bullets out of rankings. After `cortex plan spawn <slug>`, fresh plans contain template placeholders (`{{ first concrete task — link to issue/PR when filed }}`); v0.5.0 surfaced these as P0 work. v0.5.1 detects the placeholder pattern (whole-bullet `{{ ... }}`, or `{{ ... }}` segments leaving no real text) and drops them. When a plan's open items are *all* placeholders, `cortex next` emits a one-line stderr warning so the plan-still-needs-content signal isn't lost; suppressed under `--json`.

PR #69 — staleness sweep of strings the v0.5.0 release missed. `.cortex/SPEC_VERSION` written by `cortex init` was hardcoded to `0.5.0-dev`; now `0.5.0` (with `cortex.SPEC_VERSION_LITERAL` as the canonical constant). `cortex init --help` description, init scaffold `state.md` body prose, init scaffold `Incomplete:` line, `cortex status` promotion-queue absent message, and post-scaffold output all bumped from `0.4.0-dev` / "ships in v0.4.0" / "v0.6.0 lifecycle commands" to present-tense reality. Test guard added (`tests/test_init_spec_version.py`) so the next time we cut a release, the bump can't silently regress.

PR #70 — three small QoL fixes from the same dogfood:

- `cortex grep --frontmatter` excludes `.cortex/templates/` by default. Templates contain placeholder frontmatter (e.g. `Type: decision`) that produced false-positive matches on user-content searches. Opt back in with `--include-templates`; explicit `--layer templates` still works for deliberate template-library searches. `docs/grep.md` updated.
- `cortex --version` and `cortex -V` (in addition to the existing `cortex version` subcommand). Standard CLI convention.
- `cortex refresh-state` Sources line detects the project's primary dep-manifest (`pyproject.toml`, `package.json`, `Cargo.toml`, `go.mod`, `Gemfile`, `Package.swift`, `pom.xml`, `build.gradle`/`build.gradle.kts`). Renders the manifest filename and a best-effort name/version extract; falls back to `(no project manifest detected)` when none is present. New helper `src/cortex/manifests.py` so other commands (`doctor`, `status`) can reuse the detection later. autumn-mail (Swift) was the trigger — v0.5.0 reported `pyproject.toml: unknown` against a `Package.swift` project.

Conductor dispatch + monitor pattern continued to scale: three independent PRs ran in parallel from briefs in `briefs/v0.5.1-*.md`, each completing in 10–25 minutes, all merged on iteration-1 Conductor review with zero findings. Total wall time from "found bugs" to "v0.5.1 tag pushed" was approximately one hour.
