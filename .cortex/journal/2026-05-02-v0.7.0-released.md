## What shipped

Slice S1 of [`plans/cortex-retrieve.md`](../plans/cortex-retrieve.md) (sub-plan of cortex-v1.md):

- **`cortex retrieve <query> [--mode bm25] [--top-k N] [--json] [--no-rebuild]`** — new command that searches `.cortex/` markdown chunks via a SQLite FTS5 index. Default mode is `bm25` (only mode in S1; S2 will add `semantic` and `hybrid`).
- **Index storage:** `.cortex/.index/chunks.sqlite`. Gitignored, declared "hazmat" (consumers must use the CLI, not query SQLite directly). FTS5 contentless table with frontmatter preserved per chunk.
- **Markdown-aware chunking:** section-boundary first (`^## `), paragraph splits when a section exceeds the target, overlap injection between non-final chunks. 500–800 token target with 100–150 overlap per the council deltas.
- **Invalidation:** mtime+size against the working tree (NOT `git ls-tree` — council delta #1 fix; uncommitted journal edits are visible in retrieval results within one rebuild cycle).
- **Auto-rebuild hooks:** `cortex journal draft` + `cortex refresh-state` trigger an incremental rebuild iff the chunks index already exists. `cortex refresh-index --retrieve` builds both the promotion index and the chunks index. Plain `cortex refresh-index` is unchanged (only `--retrieve` triggers chunks-index work).
- **`cortex grep` untouched** (council delta #2) — verified zero sqlite/sqlite-vec/fastembed imports leak into grep paths. The zero-dep floor is preserved as doctrine.
- **Atomic writes:** new index built at `.cortex/.index/chunks.sqlite.tmp`, atomic rename on success. Prevents torn writes on interrupt.
- **Stable JSON contract:** `cortex retrieve --json` emits `[{path, score, frontmatter, excerpt}]`. This is THE file-contract Sentinel's Planner role will consume in S3 — getting the shape right was load-bearing.
- **442 lines of tests** in `tests/test_retrieve_bm25.py`: build/incremental-rebuild/invalidation-on-uncommitted-edits/file-deletion/frontmatter-preservation/section-aware-chunking/overlap/atomic-write-on-interrupt/`--no-rebuild`/JSON-schema/grep-isolation/`refresh-index --retrieve`/auto-rebuild-after-`journal draft`/latency-budget.

Acceptance verified post-merge: `cortex retrieve "doctrine canonical ownership"` on cortex's own `.cortex/` returns Doctrine 0007 hits at the top with proper BM25 scoring.

S2 (semantic + hybrid via sqlite-vec + fastembed) is a separate slice — not in this release.
