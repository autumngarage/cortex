## What shipped

- **#211** (closes #203) — `scripts/cortex-pr-merged-hook.sh` initializes `label_args=()` unconditionally before the `if gh label list ... ; then ... fi` block, so the absence of the `cortex-auto-draft` label no longer trips `set -u`'s nounset on `"${label_args[@]}"`. Regression test added covering the no-label-exists path.
- **#209** (closes #205) — `cortex` auto-sync now refuses to run when (a) `git status --porcelain` shows ANY uncommitted changes, or (b) HEAD's commit subject matches `^chore(\(.*\))?: release v\d+\.\d+\.\d+` AND the current branch isn't the default branch. Both skips emit a single-line stderr notice; neither updates the marker, so the next clean invocation still triggers a sync.

Together these close the two root-cause bugs that surfaced *during* the v1.3.0 release flow on the cortex repo itself:

- **v1.3.0 PR #201** merged cleanly but the post-merge hook crashed with `label_args[@]: unbound variable`, stranding the auto-draft journal entry on `docs/journal-pr-201`. Recovered manually as PR #202.
- **v1.3.0 push** failed initially because the just-shipped auto-sync layer regenerated `state.md` mid-push when validate ran cortex commands. Recovered by amending the regenerated files into the version-bump commit.

Both failure modes are now caught before they fire.
