## What shipped

Closes Tier 4 of [`plans/cortex-v1.md`](../plans/cortex-v1.md):

- **`cortex promote <id>` real writer** (PR #82, `3b716c9`). Replaced the validation stub with end-to-end promotion: reads candidate from `.cortex/.index.json`, writes Doctrine entry from `templates/doctrine/candidate.md` with `Promoted-from:` set, updates index, emits `Type: promotion` Journal entry. Layer-contract-respecting partial-failure handling (no destructive rollback per SPEC § 4.1/§ 4.2). Path-traversal hardening on candidate sources. 7 new regression tests covering layer-contract violations, write-order ordering, path traversal, source canonicalization, and TOCTOU on partial-write tracking.
- **`cortex doctor` invariant expansions** (PR #84, `382431e`). Nine new checks per the council-trimmed v0.6.0 scope:
  - Append-only Journal violation (SPEC § 3.5)
  - Immutable-Doctrine mutation (SPEC § 3.1) — recognizes both YAML frontmatter and `**Field:**` markdown-bold metadata, so existing cortex doctrine entries (which use the bold-inline form) parse correctly
  - Promotion-queue invariants (SPEC § 4.7)
  - CLI-less-fallback warning (Protocol § 1)
  - T1.4 file-deletion audit
  - **Generated-layer contract validation** (SPEC § 4.5) — missing seven-field provenance is an error
  - **`.cortex/config.toml` schema validation** — catches misconfigured `[audit-instructions]` blocks before `--audit-instructions` runs and silently breaks
  - **SPEC § 5.1 retention visibility** (warning only; destructive cleanup parked v1.x) — 30-day plan-archive eligibility, journal warm-tier overage
  - **Canonical-ownership warning per Doctrine 0007** — flags repo-root `ROADMAP.md` / `STATUS.md` / `PLAN.md` / `NEXT.md` / `TODO.md` as anti-pattern when `.cortex/state.md` + active plans exist; overridable via `.cortex/config.toml [doctrine.0007] allowed_root_files`
- **Doctrine 0007** — canonical ownership of "where are we" / "what's next" lives in `.cortex/`, not at repo root. Promoted from a recursive-failure dogfood incident on this same date: an agent cleaning up doc drift created `ROADMAP.md` at repo root, immediately reproducing the duplication failure mode the cleanup was supposed to fix. The recursive lesson promoted to enforceable doctrine + the v0.6.0 doctor check above.
- **Protocol bumped 0.2.1 → 0.2.2** (additive minor per SPEC § 6) — § 1 adds canonical-ownership requirement. Bundled `src/cortex/_data/protocol.md` synced so `cortex init` ships the rule to all new projects.
- **README.md `## Status` slimmed** to a one-paragraph link to `.cortex/state.md` + master plan, modeling Doctrine 0007.
