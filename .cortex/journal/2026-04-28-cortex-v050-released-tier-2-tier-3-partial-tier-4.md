## What shipped

Tier 2 — state self-maintains (was scheduled v0.4.0):

- `cortex refresh-state` — deterministic regen of `.cortex/state.md` with `<!-- cortex:hand -->` marker preservation and byte-identical idempotency under `CORTEX_DETERMINISTIC=1`.
- `cortex next` — deterministic ranked work-item list (P0/P1/P2 bands across active plan checkboxes, state.md `## Current work` + `## Open questions`, recent case studies) with stable file:line citations and `--json`. PR #58.
- `cortex plan status` — per-plan completion % and 14-day staleness flag; `--json` for scripting. PR #57.

Tier 3 — trust + automation (was scheduled v0.5.0):

- `cortex doctor --audit-instructions` — external-claim verifier that walks `CLAUDE.md` / `AGENTS.md` / `README.md` for sibling-repo, Homebrew-tap, PyPI, GitHub-release, and URL claims and checks each against reality. Configurable via `.cortex/config.toml` `[audit-instructions]`. Output never silent — clean runs report "checked N claims, all verified." PR #56.
- Per-fact `Verified:` tags (SPEC § 4.3.1, additive minor) — derived facts inside `state.md` / `doctrine/*.md` may carry an end-of-bullet `Verified: <date>` tag; `cortex manifest` surfaces stale-verification warnings inline. PR #55.
- The Touchstone post-merge hook ships separately on the Touchstone side — outside this release.

Tier 4 partial — lifecycle (was scheduled v0.6.0):

- `.cortex/.index.json` writer + `cortex refresh-index` — promotion-candidate index per SPEC § 2 / § 4.7; atomic tmp+rename writes; auto-refreshes after `cortex journal draft` and `cortex refresh-state`. Schema migrated `promote` / `manifest` / `status` reads to the new `candidates` shape with legacy `promotion_queue` fallback. PR #59.
- Real `cortex promote` writer + remaining `cortex doctor` invariant expansions (append-only Journal, immutable Doctrine, promotion-queue, single-authority, T1.4 file-deletion audit, full § 5.4 claim-trace) remain queued for v0.6.0. Briefs for both still live under `briefs/v0.6.0-T2-promote-real-writer.md` and `briefs/v0.6.0-T3-doctor-invariants.md`.

Issue-driven features:

- `cortex grep --frontmatter key:value` (closes #60) — conjunction by repetition, `!key:value` negation, `key:*` wildcard presence, list-value matching, case-insensitive keys, filter-only mode (no PATTERN), with worked Sentinel queries documented in `docs/grep.md`. PR #63.
- `cortex init --seed-from <dir>` (closes #61) — copy external Doctrine packs into `.cortex/doctrine/` preserving frontmatter byte-for-byte; smart numbering honoring the 0001-0099 reserved floor; `--merge skip-existing` for idempotent re-seed. Pairs with Sentinel's planned baseline engineering-values Doctrine pack. PR #64.

Conductor dispatch tooling validated end-to-end: PRs #63 and #64 were both implemented by `conductor exec --with codex` from `briefs/issue-60-grep-frontmatter-filter.md` and `briefs/issue-61-init-seed-from.md`, reviewed by Conductor at merge gate, and auto-merged. The dispatch-then-rebase workflow handled the README conflict on PR #64 cleanly.
