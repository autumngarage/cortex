# Task: `.cortex/.index.json` writer + `cortex refresh-index`

You are an autonomous engineering agent dispatched by Cortex to implement
Tier 4 (v0.6.0) work item T1 from `.cortex/plans/cortex-v1.md`. You operate
inside a fresh git worktree branched off `origin/main`. You will own the
implementation end-to-end and ship it via `scripts/open-pr.sh --auto-merge`.

**This task should not start until v0.4.0 has shipped.** Verify
`cortex refresh-state` and `cortex plan status` exist on main before
beginning; if either is absent, abort and report.

## Goal

Implement the `.cortex/.index.json` writer per SPEC § 2 / § 4.7. This file
is the promotion-queue's structured backing store: an index of journal
candidates that may be promoted to Doctrine. Today it does not exist; the
v0.3.0-shipped `cortex promote <id>` is a stub that exits 3 because it has
no index to read.

After this task: the index file exists and is populated; `cortex
refresh-index` rebuilds it from primary sources; `cortex journal draft`
and `cortex refresh-state` invoke the index update inline so it stays
fresh without a separate manual command.

## Required behavior

1. **JSON shape per SPEC § 2:**

   ```json
   {
     "spec": "0.5.0-dev",
     "generated": "2026-04-27T22:00:00-04:00",
     "candidates": [
       {
         "id": "2026-04-23-derive-success-from-preconditions",
         "source": ".cortex/journal/2026-04-23-derive-success-from-preconditions.md",
         "type": "decision",
         "last_touched": "2026-04-23",
         "age_days": 4,
         "tags": ["candidate-doctrine"],
         "supersedes": null,
         "promoted_to": null
       }
     ]
   }
   ```

   Atomic write: temp file + rename. Pretty-printed (indent 2) for
   readability + diff legibility.

2. **`cortex refresh-index` standalone command.** Walks
   `.cortex/journal/*.md`, identifies candidate entries (frontmatter has
   `Tags:` containing `candidate-doctrine`, OR `Type: decision` with body
   text matching configured signal patterns), emits the index. Idempotent
   on unchanged inputs.

3. **Embedded calls.** Modify `src/cortex/commands/journal.py` so
   `cortex journal draft` invokes `refresh_index()` after writing the new
   entry. Modify `src/cortex/commands/refresh_state.py` similarly. The
   invocation must be silent on success (only emit output on
   failure/warnings).

4. **Configurable signal patterns.** `.cortex/config.toml`:

   ```toml
   [refresh-index]
   candidate_patterns = [
     "we decided",
     "doctrine candidate",
     "this is load-bearing",
   ]
   ```

   Default patterns (when absent): empty (only explicit `Tags:
   candidate-doctrine` qualifies).

5. **Promoted candidates retained.** When a candidate has a
   `Promoted-to: doctrine/<slug>` field set elsewhere, the index keeps the
   row but flips `promoted_to` to the doctrine slug. The candidate stays
   in the index for audit purposes.

## Implementation outline

- New module: `src/cortex/index.py` — pure read/write of `.cortex/.index.json`.
  Functions: `read_index(path: Path) -> dict`, `write_index(path, data)`,
  `compute_index(cortex_root: Path, config) -> dict`.
- New module: `src/cortex/commands/refresh_index.py` — Click command.
- Embed in: `src/cortex/commands/journal.py`, `src/cortex/commands/refresh_state.py`.
- Existing `src/cortex/status.py` reads `.cortex/.index.json` to surface
  promotion-queue counts in bare `cortex` output — verify it now shows
  real counts after this task.

## Tests required

`tests/test_refresh_index.py`. Real `tmp_path` fixtures.

1. **Empty .cortex/journal/.** Index has `candidates: []`.
2. **Tagged candidate.** Journal entry with `Tags: [candidate-doctrine]`
   appears in candidates.
3. **Pattern-matched candidate.** Body contains `"this is load-bearing"`
   with that pattern in config — appears.
4. **Type: decision without tag or pattern match — does NOT appear.**
5. **Promoted candidate retained with `promoted_to` set.**
6. **Atomic write.** Concurrent writers don't corrupt the file (a partial
   write can't be observed; verify via injection — write a half-written
   tempfile, ensure it's not read).
7. **Idempotency.** Two `refresh-index` runs on unchanged inputs produce
   byte-identical JSON.
8. **Embedded call from `journal draft`.** After drafting an entry that
   matches config patterns, the index file contains the new candidate.
