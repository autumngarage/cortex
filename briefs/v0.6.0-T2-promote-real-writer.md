# Task: `cortex promote <id>` real writer (replace stub)

You are an autonomous engineering agent dispatched by Cortex to implement
Tier 4 (v0.6.0) work item T2 from `.cortex/plans/cortex-v1.md`. You operate
inside a fresh git worktree branched off `origin/main`. You will own the
implementation end-to-end and ship it via `scripts/open-pr.sh --auto-merge`.

**This task depends on v0.6.0 T1 (`.cortex/.index.json` writer) having
shipped.** Verify before starting; if absent, abort.

## Goal

Replace the v0.3.0-shipped stub in `src/cortex/commands/promote.py` with
end-to-end promotion: read a candidate from `.cortex/.index.json`, write a
Doctrine entry from the `doctrine/candidate.md` template, update the index
to mark the candidate promoted, and emit a `Type: promotion` Journal entry
recording the promotion event.

## Required behavior

`cortex promote <candidate-id>`:

1. **Read** the candidate from `.cortex/.index.json` via the v0.6.0 T1
   `read_index()` helper. If the candidate id is not present, exit 1
   with a clear error.

2. **Reject already-promoted.** If `candidates[id].promoted_to` is non-null,
   refuse with a clear message and exit 1. Force-promote requires `--force`
   (prompts for confirmation interactively; `--yes` skips the prompt).

3. **Generate doctrine slug** from the candidate's source journal entry.
   Naming convention: next available `NNNN-<short-slug>` where `NNNN` is
   one greater than the highest existing doctrine number, lower-bounded at
   `0100` (the 0001-0099 range is reserved for canonical entries per the
   v0.2.5 floor convention).

4. **Write the doctrine entry** at `.cortex/doctrine/<slug>.md` using the
   `doctrine/candidate.md` template. Frontmatter:
   - `Status: Active`
   - `Date: <today>`
   - `Load-priority: normal` (user can edit to `always` post-promotion)
   - `Promoted-from: journal/<candidate-source-stem>`
   - `Cites:` — copy from the source journal entry's frontmatter

5. **Update the index** to flip `promoted_to: doctrine/<slug>` for that
   candidate.

6. **Write a `Type: promotion` Journal entry** via `cortex journal draft
   promotion --no-edit` (you may need to add `promotion.md` to
   `.cortex/templates/journal/` if it doesn't exist; check first). The
   entry cites the source journal + the new doctrine entry.

7. **Output:** print the path of each new file (doctrine + journal) and
   exit 0.

8. **`--dry-run`** prints what would happen without writing.
