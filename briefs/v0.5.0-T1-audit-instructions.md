# Task: Implement `cortex doctor --audit-instructions`

You are an autonomous engineering agent dispatched by Cortex to implement
Tier 3 (v0.5.0) work item T1 from `.cortex/plans/cortex-v1.md`. You operate
inside a fresh git worktree branched off `origin/main`. You will own the
implementation end-to-end and ship it via `scripts/open-pr.sh --auto-merge`.

## Goal

Add a new `--audit-instructions` flag to `cortex doctor` that scans
`CLAUDE.md`, `AGENTS.md`, and `README.md` for claims about external artifacts
(filesystem siblings, Homebrew taps, PyPI packages, GitHub releases,
arbitrary URLs) and verifies each claim against reality. This is the active
trust-layer feature that prevents the conductor case study failure mode —
stale CLAUDE.md claim → confidently-wrong agent action.

**Output is never silent.** Even on a clean project, the command prints
`audit-instructions: checked N claims, all verified` so a clean result is
informative, not ambiguous.

## Required behavior

1. **Configurable via `.cortex/config.toml`.** New section `[audit-instructions]`
   with this schema:

   ```toml
   [audit-instructions]
   # Optional: name a Homebrew tap whose existence + current formula version
   # the audit verifies. The tap repo must exist; the current formula version
   # is matched against any `v\d+\.\d+\.\d+` reference in CLAUDE.md / README.md.
   homebrew_tap = "autumngarage/cortex"

   # Optional: list of sibling-repo path references (~ expansion supported).
   # Audit verifies each path is a directory.
   siblings = ["~/repos/touchstone", "~/repos/sentinel"]

   # Optional: PyPI package name. Audit performs an HTTPS HEAD on the index
   # to confirm the package is published.
   pypi_package = "cortex-cli"

   # Optional: list of GitHub repos whose latest release version is matched
   # against version references in CLAUDE.md / README.md.
   github_repos = ["autumngarage/cortex", "autumngarage/touchstone"]

   # Optional: list of URLs whose liveness is checked via HEAD.
   urls = ["https://github.com/autumngarage/cortex/releases"]

   # Optional: files to scan. Default: ["CLAUDE.md", "AGENTS.md", "README.md"].
   scan_files = ["CLAUDE.md", "AGENTS.md", "README.md"]
   ```

   When `.cortex/config.toml` is absent or has no `[audit-instructions]`
   section, the command runs in **discovery mode**: scans the default files,
   extracts URL references and `~/repos/<name>` references, and audits those
   only. Discovery mode never warns about missing config; it warns about
   broken claims it found.

2. **Five claim classes audited:**
   - **Filesystem siblings** — `~/repos/<name>` references in scanned files;
     verify each is a directory.
   - **Homebrew taps** — `homebrew_tap` config; run `brew tap-info --json
     <tap>` and verify it exists and is parseable.
   - **PyPI packages** — `pypi_package` config; HTTPS HEAD on
     `https://pypi.org/simple/<name>/` (allow 200/301/302).
   - **GitHub releases** — `github_repos` config; run `gh release list
     --repo <repo> --limit 1 --json tagName` and verify it returns a tag;
     match the tag against any `v\d+\.\d+\.\d+` mentions in scanned files
     and warn on mismatch.
   - **URLs** — explicit `urls` config + URLs discovered in scanned files;
     HTTP HEAD with 5-second timeout; allow 2xx/3xx; warn on 4xx/5xx/timeout.

3. **Output shape.** Default human output to stdout:

   ```
   audit-instructions:
     ✓ filesystem sibling: /Users/henry/repos/touchstone exists
     ✓ homebrew tap: autumngarage/cortex (formula at v0.3.0)
     ⚠ github release version mismatch: CLAUDE.md mentions v0.2.5, latest is v0.3.0
     ✓ url: https://github.com/autumngarage/cortex/releases (200)
     ⚠ filesystem sibling: ~/repos/old-repo missing (CLAUDE.md:42)
   audit-instructions: checked 5 claims, 2 warnings
   ```

   `--json` emits a machine-readable summary.

4. **Exit codes.** Default: warnings exit 0 (informational). Under `--strict`,
   warnings exit 1 so the future Touchstone pre-push hook can block.

5. **Performance.** All network checks parallel via `concurrent.futures`
   thread pool (size 8). Total wall time on a typical project must be under
   10 seconds even with several URL checks.

6. **Two non-goals** carried from the case study (do not implement):
   - Does not police narrative/principles prose ("we use TDD" claims, etc.).
   - Does not require release-per-commit hygiene.

## Implementation outline

- New module: `src/cortex/audit_instructions.py` — pure scanning + verification
  logic. Each claim class is a function that takes config + scanned text and
  returns a list of `Finding` dataclass instances (`level`, `message`,
  `source_file`, `line_number`).
- Wire into `src/cortex/commands/doctor.py` — add `--audit-instructions` and
  `--strict` flags (the existing doctor command takes `--audit` and
  `--audit-digests`; this is the third audit flag).
- Config parsing: extend the existing `.cortex/config.toml` reader (grep for
  it; if no shared loader exists, create `src/cortex/config.py` and migrate
  the existing readers in a separate commit if cheap).
- Network checks use `urllib.request.Request(method='HEAD')` with a 5s
  timeout; do not add `requests` as a dependency.
- `gh` and `brew` invocations via `subprocess.run` with explicit timeout;
  graceful degradation when CLIs are absent (warn once: "gh not installed,
  skipping github_repos checks").

## Tests required

Place tests under `tests/test_audit_instructions.py`. Use real `tmp_path`
fixtures for the project tree; mock only the network/subprocess boundary
(use `monkeypatch` on `urllib.request.urlopen` and `subprocess.run`).

Required test cases:

1. **Clean project, no findings.** Fixture with config + valid claims;
   monkeypatch network/subprocess to return success; output is exactly
   `audit-instructions: checked N claims, all verified` with N matching the
   claim count.
2. **Stale Homebrew formula version.** Mocked `brew tap-info` returns
   v0.3.0; CLAUDE.md mentions v0.2.5 — finding emitted with file path + line
   number.
3. **Missing sibling.** Config names `~/repos/missing`; warn with the
   reference's source line.
4. **Discovery mode.** No config; CLAUDE.md mentions `~/repos/touchstone`
   (existing) — that gets audited; output reports it verified.
5. **URL 404.** Mocked HEAD returns 404; warning emitted.
6. **`--strict` exits 1 on any warning.**
7. **`--json` shape parses.**
8. **Always-prints-summary.** Even with zero findings on zero configured
   claims (empty config), prints `checked 0 claims, all verified`.
9. **Network timeout treated as warning, not crash.** Mocked timeout
   raises; finding emitted; doctor exits 0 (no `--strict`).
10. **`brew` / `gh` absent gracefully degrades** (PATH-strip via
    `monkeypatch.setenv`).

## Acceptance criteria

- `cortex doctor --audit-instructions` runs from this repo's root and
  produces a non-empty, non-silent report.
- `--strict` is supported on `cortex doctor`.
- All existing tests pass; new tests pass under

- `cortex doctor` without `--audit-instructions` continues to behave
  exactly as before.

  --audit-instructions` on this repo as a sanity check.
