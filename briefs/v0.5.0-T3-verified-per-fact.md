# Task: Manifest `Verified:` per-fact (SPEC § 4.3 extension)

You are an autonomous engineering agent dispatched by Cortex to implement
Tier 3 (v0.5.0) work item T3 from `.cortex/plans/cortex-v1.md`. You operate
inside a fresh git worktree branched off `origin/main`. You will own the
implementation end-to-end and ship it via `scripts/open-pr.sh --auto-merge`.

## Goal

Extend SPEC § 4.3 (Generated layers declare provenance) so derived facts
inside `state.md` and `doctrine/*.md` can carry a `Verified: <date>` tag on
individual bullets — not just the whole-file `Generated:` timestamp. Then
extend `cortex manifest` to surface stale `Verified:` warnings inline next
to the fact itself.

This is the **passive freshness signal** that pairs with the active
`--audit-instructions` check (separate task). Together they make stale
external-artifact claims structurally hard to ship: the audit catches them
when the user runs the check; the inline `Verified:` warning catches them
when an agent reads the manifest at session start.

## SPEC amendment required

Read the current `SPEC.md` § 4.3. The amendment adds a new sub-section
§ 4.3.1 "Per-fact verification timestamps":

```markdown
### 4.3.1 Per-fact verification timestamps (additive)

A bullet inside a generated layer (`state.md`, `doctrine/*.md`, digest
entries) may carry a trailing `Verified: <ISO-8601 date>` tag indicating
when the asserted fact was last verified against its source of truth.
The tag is invisible to consumers that don't parse it (it's plain
markdown text). Consumers that do parse it (e.g., `cortex manifest`)
surface a warning when `today - Verified:` exceeds a project-configured
staleness threshold (default 90 days).

Shape:

    - The Homebrew tap is `autumngarage/cortex`. Verified: 2026-04-26

The `Verified:` marker MUST appear at end-of-bullet. Multi-line bullets
keep the marker on the last line. The date is ISO-8601 (`YYYY-MM-DD`);
a full timestamp is also accepted but the date portion is what's compared.

This is an additive amendment per § 7 (compatibility); SPEC bumps from
v0.4.0-dev to v0.5.0-dev.
```

Apply this amendment in the same PR. Bump `SPEC.md` version to
`v0.5.0-dev`. Update `SUPPORTED_SPEC_VERSIONS` in `src/cortex/__init__.py`
to include `0.5`.

## Implementation outline

- New module: `src/cortex/verified.py` — pure parsing + age computation.
  Functions:
  - `parse_verified(line: str) -> date | None` — extracts the date or returns
    None for un-tagged bullets.
  - `bullet_age_days(verified: date, today: date | None = None) -> int`
  - `format_warning(age_days: int, threshold: int) -> str | None` —
    returns `"⚠ verified {age}d ago"` or None if fresh.
- Manifest integration: `src/cortex/manifest.py` — when rendering Doctrine
  + state.md slices, parse each bullet's `Verified:` tag, compute age, and
  emit the warning **inline** at end-of-bullet *before* the `Verified:`
  tag itself (so the warning is visible to the agent reading the manifest).
- Config: extend `.cortex/config.toml` with `[manifest]` section:

  ```toml
  [manifest]
  verified_threshold_days = 90  # default
  ```

  When config absent, default to 90.

## Tests required

Place tests under `tests/test_verified.py`. Use real `tmp_path` fixtures.

Required test cases:

1. **Parser — well-formed.** `"- Foo. Verified: 2026-04-26"` → date(2026, 4, 26).
2. **Parser — full timestamp.** `"- Bar. Verified: 2026-04-26T12:00:00-04:00"`
   → date(2026, 4, 26).
3. **Parser — no marker.** `"- Plain bullet"` → None.
4. **Parser — multi-line bullet.** `"- Top line\n  more text\n  Verified: 2026-04-26"`
   → date(2026, 4, 26).
5. **Age computation.**
6. **Manifest integration — fresh bullet renders unchanged.**
7. **Manifest integration — stale bullet gets inline warning prefix.**
8. **Manifest integration — threshold respected from config.**

Plus existing manifest tests must still pass.
