# Task: cortex next — drop `{{ placeholder }}` strings from rankings

You are an autonomous engineering agent dispatched by Cortex to address
a finding from the autumn-mail dogfood of v0.5.0 (2026-04-28). You
operate inside a fresh git worktree branched off `origin/main`. You
will own the fix end-to-end and ship via `scripts/open-pr.sh
--auto-merge`.

## Goal

After `cortex plan spawn <slug>`, the freshly-scaffolded plan contains
template placeholders like `- [ ] {{ first concrete task — link to
issue/PR when filed }}`. `cortex next --json` then surfaces these
placeholder strings as P0 ranked work items. The deterministic ranker
should ignore checkbox/bullet items whose visible text is dominated by
unfilled `{{ ... }}` placeholders — they are scaffolding, not work.

## Required behavior

`cortex next` and `cortex next --json` (in `src/cortex/ranking.py`)
must filter out items whose post-cleanup text matches the placeholder
pattern.

1. **Definition of placeholder.** A bullet/checkbox item is a
   placeholder when its cleaned text:
   - Is wholly enclosed in `{{` ... `}}` (e.g. `{{ TODO }}`), OR
   - Contains a `{{ ... }}` segment that, when removed, leaves an
     empty or whitespace-only string, OR
   - Contains a `{{ ... }}` segment AND no alpha-numeric characters
     outside of any `{{ ... }}` segments. (This catches `- [ ] {{ x }}`
     and `- [ ] {{ x }}` followed by trailing whitespace/punctuation
     but allows `- [ ] Real task with {{ optional placeholder }}`.)

2. **Filter in `_open_plan_checkboxes` and `_section_items`.** Drop
   placeholder items before they enter any band. `_recent_case_studies`
   does not need filtering (case-study summaries are first-paragraph
   text, not bullets).

3. **The filter does NOT modify input files.** This is a read-side
   filter only.

4. **Surface a notice when a plan has only placeholder items.** When
   `cortex next` finishes building bands, if a plan has open
   checkboxes that all got filtered out, emit a one-line warning to
   stderr: `warning: plans/<slug>.md has open items but all are
   {{ placeholder }} — fill in concrete tasks or check off the stubs.`
   This keeps the user aware that the plan exists but is empty in
   practice (won't surface in the human/JSON output, but the
   underlying `plans/<slug>.md` still has open boxes that doctor /
   plan status will report on). Suppressed under `--json` (machine
   consumers shouldn't get human-text on stderr by default — but
   `--json` consumers can re-derive via `cortex plan status` if they
   need it).

## Implementation outline

- Add `is_placeholder_text(text: str) -> bool` to
  `src/cortex/ranking.py` (or a small new helper module if you'd
  rather not grow `ranking.py`).
- The function operates on the text AFTER `_clean_item_text` has run
  (so markdown formatting is stripped). Detection rule per (1) above.
- Plumb the filter into `_open_plan_checkboxes` (for active-plan
  bullets) and `_section_items` (for state.md bullets).
- Track the *names of plans* that had all-filtered output during
  `collect_next_items` and emit the stderr warning from
  `run_next` / the Click command body — not from `collect_next_items`
  itself (keep the collection function pure).

## Tests required

Add to `tests/test_next.py`:

1. **Plan with placeholder items not in P0.** Spawn a plan via the
   `_write_plan` fixture with work_items containing `- [ ] {{ task }}`;
   assert the corresponding entry does not appear in `data["p0"]`.
2. **Plan with mixed real + placeholder items.** Real items appear in
   P0; placeholder ones don't.
3. **State.md with placeholder bullet under `## Open questions`.**
   Same — placeholder gets filtered.
4. **Plan with ONLY placeholders triggers stderr warning** (use
   `CliRunner` with `mix_stderr=False`). Warning text must mention the
   plan path. Under `--json`, the warning is suppressed.
5. **`is_placeholder_text` unit tests** — at least these inputs:
   - `"{{ task }}"` → True
   - `"{{ first concrete task — link to issue/PR when filed }}"` → True
   - `"Implement retry logic with {{ optional flag }}"` → False
   - `"plain task"` → False
   - `""` → False (empty isn't a placeholder; existing parsing
     prevents empty entries from arriving)
   - `"  "` → False
   - `"{{ }} {{ }}"` → True (multiple empty placeholders, no real text)
   - `"foo{{x}}bar"` → False (real text outside the placeholder)
