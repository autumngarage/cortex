# Task: Remaining `cortex doctor` invariant expansions

You are an autonomous engineering agent dispatched by Cortex to implement
Tier 4 (v0.6.0) work item T3 from `.cortex/plans/cortex-v1.md`. You operate
inside a fresh git worktree branched off `origin/main`. You will own the
implementation end-to-end and ship it via `scripts/open-pr.sh --auto-merge`.

**This task depends on v0.6.0 T1 (`.cortex/.index.json` writer) for the
promotion-queue invariant check; verify T1 has shipped before starting.**

## Goal

Expand `cortex doctor` to enforce every SPEC invariant that's currently
unchecked. After this task, every contract in SPEC.md has either a doctor
check or a documented reason for staying advisory.

## Required behavior — six new checks

For each check below, add a function in `src/cortex/doctor_checks.py`
(creating that module if needed; consolidate existing doctor logic into
it). Each check takes a `Path` and returns a list of `Finding` instances.

### 1. Append-only Journal violation (SPEC § 3.5)

Walk `git log --follow .cortex/journal/<file>.md` for every journal file.
If any commit modifies an existing journal file (vs. adding it), emit a
warning: `journal <file> modified at commit <sha> — append-only invariant
violated`. The check is best-effort: a file moved (rename) does not count
as a modification. Exception: changes to YAML frontmatter `Updated-by:`
are permitted by SPEC § 3.5; detect frontmatter-only diffs and skip.

### 2. Immutable-Doctrine mutation (SPEC § 3.1)

Same shape as #1, but for `.cortex/doctrine/*.md`. Doctrine entries are
immutable except via Superseded-by. A non-superseded entry that's been
modified post-creation is a violation. Allow `Status:` field changes
(Active → Superseded-by Nxxx) and frontmatter-only diffs.

### 3. Promotion-queue invariants (SPEC § 4.7)

Read `.cortex/.index.json` via T1's helper. Verify:
- Every `candidates[].source` resolves to a real file under `.cortex/journal/`.
- Every `candidates[].promoted_to` (when set) resolves to a real file under
  `.cortex/doctrine/`.
- No two candidates share the same `id`.
- `age_days` matches `today - last_touched` within 1 day.

### 4. Single-authority drift (SPEC § 4.8)

For every layer marked `Generator: cortex refresh-state` (etc.) in
frontmatter, verify the file's mtime is within `Generated:` ± 5 minutes
(allowing for filesystem timestamp slop). A drift means someone hand-edited
a generated layer outside the `<!-- cortex:hand -->` markers — warn.

### 5. CLI-less-fallback warning (Protocol § 1)

If the project lacks a `.cortex/.index.json` AND has more than 20 Doctrine
entries OR more than 100 Journal entries, warn that the corpus exceeds the
fallback threshold and `cortex refresh-index` should run.

### 6. T1.4 file-deletion audit

Walk `git log` for the audit window (default 7 days). For every commit that
deleted a file with more than the configured threshold (default 100 lines)
that isn't matched by a `Type: decision` Journal entry within 72 h of the
commit, emit a warning. The threshold is configurable via
`.cortex/protocol.md` per-project override (already documented in
Protocol § 2). Reuse the existing audit window logic in
`src/cortex/audit.py`.

### 7. § 5.4 claim-trace audit (full implementation)

The current `--audit-digests` is first-slice — it samples bullets and
warns when most lack citations. Replace with the full SPEC § 5.4 check:
for every claim in a digest, verify it traces to at least one source
journal entry. A citation is a markdown link or path reference resolving
to a `.cortex/journal/` file. Sampling is replaced by exhaustive
verification.

## Implementation outline

- New module: `src/cortex/doctor_checks.py` consolidates check functions.
- Wire into `src/cortex/commands/doctor.py`. Each check is gated by either
  the existing `doctor` invocation or the existing `--audit` /
  `--audit-digests` / new `--audit-instructions` flags as appropriate.
  - #1, #2, #4 run on plain `cortex doctor` (always).
  - #3, #6 run under `--audit`.
  - #5 runs on plain `cortex doctor` (always — surface on every invocation).
  - #7 replaces the `--audit-digests` first-slice.
- All new checks emit warnings (exit 0); under `--strict`, warnings exit 1.

## Tests required

`tests/test_doctor_invariants.py`. Real `tmp_path` git fixtures.

1. **Append-only violation detected.** Commit + then modify a journal file;
   doctor warns.
2. **Frontmatter-only Updated-by diff allowed.** Same shape but only
   frontmatter changed; doctor does not warn.
3. **Doctrine mutation detected (non-supersede).** Commit a doctrine entry,
   modify body — warns. Same with supersede status change — does not warn.
4. **Promotion-queue dangling source warns.**
5. **Single-authority drift detected** (generated file mtime > Generated +
   5min).
6. **CLI-less-fallback threshold met — warns.**
7. **T1.4 deletion without journal — warns.**
8. **Claim-trace failure — warns.** Digest cites no journal entry; warning
   emitted.
